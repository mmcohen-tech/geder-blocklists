! Title: Geder - DoH / DoT Anti-Bypass Blocklist
! Description: Blocks DNS resolution for known DNS-over-HTTPS (DoH) and
!              DNS-over-TLS (DoT) endpoints. Required companion to the
!              port-53 firewall intercept — without this, modern browsers
!              bypass the intercept via encrypted DNS on port 443.
! Maintained by: Geder
! Last updated: 2026-04-19
! Format: AdGuard / AdBlock syntax. ||domain^ blocks domain + all subdomains.
!
! HOW THIS WORKS:
! When a browser tries to use DoH, it first needs to resolve the DoH
! endpoint's hostname (e.g., "dns.google"). By blocking that lookup at
! AdGuard, the browser's DoH fails → it falls back to system DNS → the
! port-53 firewall rule catches it → AdGuard filters normally.
!
! MAINTENANCE: DoH endpoints evolve slowly. Review quarterly. Watch for:
!  - New entries in browser DoH provider lists (Chrome, Firefox, Edge)
!  - New public DoH providers launching
!  - VPN services adding DoH (common in 2024-2026)
